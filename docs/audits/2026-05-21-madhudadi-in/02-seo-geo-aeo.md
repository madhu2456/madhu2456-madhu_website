## GEO Findings

### GEO-001 - Portfolio machine-readable endpoints are stale and not fully aligned with the blog

Confidence: Verified  
Severity: Medium

Evidence:

- Runtime `https://madhudadi.in/llms.txt` reports last content update `2026-05-03T20:00:00.000Z`.
- Runtime `https://madhudadi.in/ai-profile.json` reports `meta.dateModified: 2026-05-03T20:00:00.000Z`.
- Runtime `https://madhudadi.in/blog/ai-profile.json` reports `lastContentUpdate: 2026-05-19T12:11:17.817152Z`.
- Runtime blog AI profile reports `totalPublishedPosts: 25`, `totalSeries: 2`, `totalTags: 12`; portfolio AI profile only gives a generic blog object and does not include the same content inventory.
- Blog author role appears as `AI Architect, Data Analyst, Python Engineer`; portfolio headline appears as `AI Developer & Marketing Analytics Leader`.
- Blog location includes `Visakhapatnam, Andhra Pradesh, India`; portfolio location is `Visakhapatnam, India`.
- Source `src/app/llms.txt/route.ts:185` uses `portfolioLastUpdatedAt`.
- Source `src/app/ai-profile.json/route.ts:259` uses `portfolioLastUpdatedAt`.

Impact:

- AI systems can retrieve two related first-party machine-readable profiles with different freshness dates and entity descriptors.
- The portfolio endpoint under-represents current blog content, which weakens entity consolidation and citation confidence for blog topics.

Fix:

- Add a first-party blog summary import/build step into portfolio `llms.txt` and `ai-profile.json`.
- Align canonical author role, location granularity, sameAs, expertise labels, and latest content stats across portfolio and blog.
- If the blog app remains a separate codebase, consume its live `/blog/ai-profile.json` during a controlled build/import step and store the normalized subset in portfolio data.

### GEO-002 - Root `llms-full.txt` is advertised but missing

Confidence: Verified  
Severity: Low

Evidence:

- Live robots AI crawler group includes `Allow: /llms-full.txt`.
- `https://madhudadi.in/llms-full.txt` returns 404.
- `https://madhudadi.in/blog/llms-full.txt` returns 200.

Fix:

- Either implement `src/app/llms-full.txt/route.ts` for a root full profile or remove the root allow hint from live robots.
